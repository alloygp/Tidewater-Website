02 Security

Security is built in at every layer — from encrypted connections to spam filtering on forms to how sensitive keys are stored. Nothing confidential lives in the website's code.

Encrypted everywhere (HTTPS)
Every page and form is served over an encrypted connection with an automatically renewed SSL certificate. There's nothing to manage and it never expires unexpectedly.
Browser protection headers
The site sends security headers (clickjacking protection, content-type locking, and referrer privacy) that harden it against common web attacks.
Spam & bot filtering
Forms use a hidden “honeypot” trap that silently catches automated bot submissions — so your inboxes stay clean — without making real visitors solve a puzzle.
Secrets kept out of the code
API keys and credentials are stored as encrypted environment variables on the host — never written into the website's code or repository.
Server-side validation
Every submission is checked on the server, not just in the browser, so required fields and valid email addresses are confirmed before anything is sent.
Failure alerts
If a form email ever fails to send, our team is alerted immediately so we can step in — no submission silently disappears.

03 How to request changes & updates

Whether it's a copy tweak, a new page, or a change to who receives form submissions, the process is the same. Send it to us and we'll handle the rest.

1
Send your request to Alloy GP
Email admin@alloygp.co (or your usual contact) with what you'd like changed. A screenshot or the page link is always helpful.
2
Include the details
Tell us the page or section, the exact new wording or image, and the deadline if there is one. The more specific, the faster we can turn it around.
3
We make the change & preview it
We update the site and, for anything substantial, share a preview link so you can review it before it goes live.
4
You approve, we publish
Once you sign off, the change publishes automatically. Every update is version-controlled, so it can be rolled back if needed.
