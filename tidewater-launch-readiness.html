Set PUBLIC_ENV=production on the Vercel production project before launchThe noindex logic checks for an env var called PUBLIC_ENV. Any deployment where this variable is not set to production gets noindexed automatically — which is correct for staging, but will also noindex the live site if you forget this step. Before pointing the domain: Vercel → production project → Settings → Environment Variables → add PUBLIC_ENV = production, then redeploy. After that: production (has the variable) → indexed normally. Staging (no variable) → noindexed automatically. You don't need to do this until launch day, but it is a hard requirement — skip it and the live site will be invisible to Google.
